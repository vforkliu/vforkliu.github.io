游戏安全：月影外挂透视功能分析

发表于 2019-07-06 | 更新于 2019-07-07 | 分类于 游戏安全 | 评论数：

简介

透视功能是FPS游戏面临的主要威胁之一：它可以始终显示目标的位置，极大地破坏了游戏的乐趣。实现透视涉及到两个关键技术点：目标位置计算和标注目标。

标注目标

标注目标一般画一个小方框。本文的月影外挂就是画一个小红方框。
月影外挂采用clear函数绘制小红框：

这是DirectX里的一个函数，参数含义如下：

  • Count：说明你要清空的矩形数目。如果要清空的是整个客户区窗口，则设为0；
  • pRects：这是一个D3DRECT结构体的一个数组，如果count中设为5，则这个数组中就得有5个元素。
  • Flags：一些标记组合。只有三种标记：D3DCLEAR_STENCIL , D3DCLEAR_TARGET , D3DCLEAR_ZBUFFER。
  • Color：清除目标区域所使用的颜色。
  • Z：设置Z缓冲的Z初始值。小于或等于这个Z初始值的Z值才会被改写，但它的值只能取0到1之间。
  • Stencil：设置模板缓冲的初始值。它的取值范围是0到2的n次方减1。其中n是模板缓冲的深度。 上图显示了一次Clear调用过程，画一个方框共调用四次。

目标位置计算

在3D游戏里每个游戏对象都有一个位置的成员变量，这个变量存放的是世界坐标系下的坐标。在FPS游戏里，为了游戏的流畅度和减小开发难度，敌对阵营所有目标的位置坐标在每时每刻都可以在客户端获取。或者说游戏服务器没有建立起一个三维世界，游戏服务器不知道某个游戏对象是否被遮挡，如果知道，理论下就不会下发这个游戏对象的位置信息。这是网络游戏某些逻辑呈现本地化的重要例子。

世界坐标转屏幕坐标

假设已经获取了某个目标的世界坐标，如何将世界坐标转换为屏幕坐标呢？先来看一个关键函数：

  • pOut 结果向量
  • pV 源向量
  • pViewport 视口
  • pProjection 投影矩阵
  • pView 视变换矩阵
  • pWord 世界变换矩阵
    这个函数就可以将世界坐标系的坐标转换为屏幕坐标系的坐标。

获取世界坐标

游戏客户端一般会有一个全局指针，保存了所有目标的全部信息。关键点：找到这个全局数据结构的位置。

检测方法

位置防御采用坐标加密：设计一套加密算法，对位置坐标机密，而且可以线上替换加密算法。

目标标注：根据外挂标注的方法，做对应的检测，比如检测外挂是否调用了某个函数。

图像识别：通过截屏的方式，将玩家游戏过程的游戏屏幕上传到服务器，可以采用人工标注或图像识别技术检测玩家是否作弊。
